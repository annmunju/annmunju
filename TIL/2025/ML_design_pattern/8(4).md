## 사용 사례와 데이터 유형에 따른 일반적인 패턴

### 1. 자연어 이해
> 텍스트 분류(이메일 필터링), 엔티티 추출, 질문 응답, 음성 인식, 텍스트 요약, 감정 분석 등
- 임베딩 : 텍스트를 벡터로 변환하여 의미적 유사성을 파악
- 특징 해시 : 고차원 텍스트 특징을 저차원으로 효율적으로 매핑
- 중립 클래스 : 분류 불가한 경우를 위한 별도 클래스 도입
- 멀티모달 입력 : 텍스트 외 이미지, 음성 등 다양한 입력 결합
- 전이 학습 : 사전 학습된 모델을 활용해 적은 데이터로 성능 향상
- 2단계 예측 : 1차 예측 후 추가 모델로 정밀 예측 수행
- 캐스케이드 : 여러 모델을 순차적으로 적용해 효율적 예측
- 윈도 추론 : 텍스트의 일정 구간(윈도우) 단위로 예측 수행

### 2. 컴퓨터 비전
> 이미지 분류, 비디오 동작 분석, 이미지 분할, 이미지 노이즈 제거 등
- 리프레이밍 : 문제를 다른 형태로 변환하여 해결
- 중립 클래스 : 분류가 어려운 경우를 위한 별도 클래스 도입
- 멀티모달 입력 : 이미지 외 텍스트, 음성 등 다양한 입력 결합
- 전이 학습 : 사전 학습된 모델을 활용해 적은 데이터로 성능 향상
- 임베딩 : 이미지를 벡터로 변환하여 의미적 유사성 파악
- 멀티라벨 : 하나의 이미지에 여러 라벨을 부여하는 분류
- 캐스케이드 : 여러 모델을 순차적으로 적용해 효율적 예측
- 2단계 예측 : 1차 예측 후 추가 모델로 정밀 예측 수행

### 3. 예측 분석
>  에너지 예측, 고객 이탈 예측, 재무 모델링, 날씨 예측, 예측 유지 보수 등
- 특징 교차 : 여러 특징을 조합하여 새로운 정보를 생성
- 특징 저장소 : 재사용 가능한 특징을 중앙에서 관리
- 임베딩 : 범주형/연속형 데이터를 저차원 벡터로 변환
- 앙상블 : 여러 모델의 예측을 결합해 성능 향상
- 트랜스폼 : 입력 데이터를 모델에 맞게 변환
- 리프레이밍 : 문제를 다른 형태로 변환하여 해결
- 캐스케이드 : 여러 모델을 순차적으로 적용해 효율적 예측
- 멀티라벨 : 하나의 샘플에 여러 라벨을 부여하는 분류
- 중립 클래스 : 분류가 어려운 경우를 위한 별도 클래스 도입
- 윈도 추론 : 일정 구간(윈도우) 단위로 예측 수행
- 배치 서빙 : 여러 예측을 한 번에 처리하여 효율성 향상

### 4. 추천시스템
> 추천 시스템은 과거 사용자들의 활동에서 유사한 특징을 포착하고 새롭게 주어진 사용자와 가장 관련성이 높은 항목을 추천
- 임베딩 : 사용자/아이템을 벡터로 변환해 유사성 파악
- 앙상블 : 여러 모델의 예측을 결합해 추천 성능 향상
- 멀티라벨 : 하나의 사용자에게 여러 추천 항목 제공
- 전이 학습 : 사전 학습된 모델을 활용해 적은 데이터로 성능 향상
- 특징 저장소 : 재사용 가능한 특징을 중앙에서 관리
- 특징 해시 : 대용량 범주형 특징을 효율적으로 매핑
- 리프레이밍 : 문제를 다른 형태로 변환해 추천 문제 해결
- 트랜스폼 : 입력 데이터를 추천 모델에 맞게 변환
- 윈도 추론 : 일정 구간(윈도우) 단위로 추천 수행
- 2단계 예측 : 1차 추천 후 추가 모델로 정밀 추천 수행
- 중립 클래스 : 추천 불가한 경우를 위한 별도 클래스 도입
- 멀티모달 입력 : 텍스트, 이미지 등 다양한 입력 결합
- 배치 서빙 : 여러 추천을 한 번에 처리해 효율성 향상
